响应式

在项目中深层级的响应式传递有两种方式

  1. project/inject

  2. store

  3. projecy / inject
    定义一个 const activePanelBodyItem = ref()
    通过project传进去，响应式保持
    如果是一个reactive对象，结构时需要用toRefs包裹，维持响应性

  4. store
    问题解释：
    直接访问 store.activePanelBodyItem 的问题：
    当你访问 store.activePanelBodyItem 时，Vue 的响应式系统无法正确追踪这个嵌套的 ref
    这会导致 computed 不会在 activePanelBodyItem 变化时重新计算
    storeToRefs 的作用：
    storeToRefs 会将 store 中的响应式属性提取出来，保持它们的响应式连接
    这样 activePanelBodyItem 就是一个独立的 ref，可以被 Vue 的响应式系统正确追踪

解构的方式：const { activePanelBodyItem } = useDataBoardStore()
虽然这种方式有时候能工作，但 storeToRefs 是更可靠的方式

响应式丢失的问题主要出现在：
解构 reactive 对象的属性时
解构 Pinia store 时（需要用 storeToRefs）
