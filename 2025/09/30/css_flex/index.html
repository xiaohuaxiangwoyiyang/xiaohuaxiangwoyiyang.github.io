Flex

1. flexBasis

flexBasis 定义了 Flex 项目在主轴方向上分配多余空间之前（或者在空间不足时收缩之前）的初始大小
flex-basis 的常用值

auto (默认值)

意思是“请参考我的 width 或 height 属性”。

如果项目没有设置 width/height，则大小由项目的内容决定（内容撑开）。

长度值 (如 100px, 20%, 10em)

直接设置一个固定的基准尺寸。例如 flex-basis: 200px 意味着项目的初始大小为 200 像素。

content

根据项目的内容自动决定大小，完全忽略 width / height 属性。这在你希望尺寸完全由内容决定时非常有用。

0

将项目的初始大小设置为 0。这意味着所有的空间（无论是正还是负）都将由 flex-grow 和 flex-shrink 来分配。这在实现等分布局时非常常用。
flexBasis 主要控制主轴方向的尺寸（在水平布局中是宽度）。但固定高度影响的是：
视觉体验：固定高度的子元素在父容器变小时可能溢出或显示异常
布局稳定性：响应式高度让整个面板更协调地调整
用户体验：当面板宽度变化时，用户期望看到所有内容都相应调整
修改为 height: 100% 的主要好处是：
子元素会完全适应父容器的尺寸变化
视觉上更一致和流畅
避免了固定尺寸可能带来的布局问题
所以虽然固定高度不会直接”阻止” flexBasis 工作，但会影响整体的用户体验和视觉效果。

2. max-height: 390px 导致按钮元素移除屏幕外的原因是：

Flexbox 布局冲突：.comment 区域设置了 flex: 1，会尽可能占据空间
内容溢出：当评论内容（特别是图片）过多时，会超出 max-height 限制
父容器限制：父容器设置了 overflow-y: hidden，导致被压缩的底部区域不可见
解决方案：
给 .comment 添加 min-height: 0 和 overflow-y: auto，让内容区域可以滚动
给 .footer 添加 flex-shrink: 0，防止按钮区域被压缩

3. 一个需求，在大屏幕上一行显示，在小屏幕上两行显示，如何做？

工作原理：

大屏幕时：父容器使用 justify-content: space-between，自动将 title-left 推到最左边，title-right 推到最右边
小屏幕时：
父容器改为 flex-direction: column（垂直排列）
title-left 使用 justify-content: flex-start（左对齐）
title-right 使用 justify-content: flex-end（右对齐）

align-items: stretch 的含义

align-items: stretch 是 CSS Flexbox 布局中的一个属性，用于控制交叉轴（垂直于主轴）上子元素的排列方式。
具体作用：
拉伸子元素：让所有子元素在交叉轴方向上拉伸到父容器的完整高度
默认行为：这是 align-items 的默认值，当没有显式设置时，子元素会自动拉伸

在这个代码上下文中的作用：

当屏幕宽度小于 1620px 时：
flex-direction: column 将布局从水平改为垂直
align-items: stretch 确保 title-left 和 title-right 两个子元素都拉伸到容器的完整宽度

觉效果对比：
不使用 stretch（默认 flex-start）：
┌─────────────────┐
│ [title-left] │ ← 只占用内容宽度
│ [right]│ ← 只占用内容宽度
└─────────────────┘

使用 stretch：
┌─────────────────┐
│ [title-left ]│ ← 拉伸到完整宽度
│ [ right ]│ ← 拉伸到完整宽度
└─────────────────┘

实际意义：
在这个响应式设计中，stretch 确保了在小屏幕下，左右两个区域都能充分利用可用空间，提供更好的视觉平衡和用户体验

什么是 flex-shrink？

flex-shrink 是 Flexbox 布局中的一个属性，用于控制当 Flex 容器空间不足时，Flex 项目如何收缩。它决定了项目在空间不够时的”收缩意愿”。

核心概念

  • 作用时机：只在 Flex 容器空间不足时生效

  • 默认值：1（允许收缩）

  • 取值：数字（通常是非负整数），数值越大收缩得越多

  • 工作原理：项目根据其 flex-shrink 值和 flex-basis 的乘积权重来按比例收缩

总结

  • flex-shrink 是控制 Flex 项目收缩行为的关键属性：

  • flex-shrink: 0：禁止收缩，保持基准大小

  • flex-shrink: 1：默认值，允许等比例收缩

  • flex-shrink: >1：收缩意愿更强，会比普通项目收缩更多
